## Onboarding Progress Indicator

```jsx
// 5-step progress dots
<nav aria-label="Onboarding progress">
  {[1,2,3,4,5].map(step => (
    <span
      key={step}
      aria-current={step === currentStep ? 'step' : undefined}
      aria-label={`Step ${step}${step < currentStep ? ' (completed)' : step === currentStep ? ' (current)' : ''}`}
      className={step <= currentStep
        ? 'w-2.5 h-2.5 rounded-full bg-primary-600'
        : 'w-2.5 h-2.5 rounded-full bg-neutral-200'
      }
    />
  ))}
</nav>
```

## Onboarding UX Principles

1. **One action per screen** — never overwhelm with multiple decisions
2. **Always provide an escape** — every step has a skip/cancel option
3. **Show progress** — users know where they are and how far to go
4. **Explain the why** — wallet connect must explain benefits
5. **Celebrate completion** — confetti + personalized success message
6. **Persist progress** — save partial state so users can resume
7. **Mobile first** — all screens designed for 375px viewport
